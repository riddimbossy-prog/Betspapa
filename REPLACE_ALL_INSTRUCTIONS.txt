BETSPAPA FULL REPOSITORY REPLACEMENT — v1.13.0
================================================

IMPORTANT
---------
Do not deploy the older v1.11 ZIP. This package preserves its account features
and adds the amended Papa's Pick engine.

DEPLOY CODE
-----------
1. Extract this ZIP.
2. Replace every file in your Betspapa repository.
3. Commit:
   BetsPapa v1.13 amended Papa's Pick
4. Push origin from GitHub Desktop.
5. Wait for GitHub Pages and Render.
6. Confirm:
   https://api.betspapa.com/api/health
   Service version must show 1.13.0.
   Engine version must show papasense-v1.13.0.

PAPA'S PICK v1.13
-----------------
- Team Over 0.5 odds below 1.20 are rejected.
- Only the same team's Over 1.5 may replace it.
- The replacement must pass every statistical and value gate.
- When it fails, no forced upgrade is made.
- 1/1 and 2/2 map to the relevant Team to Win Either Half.
- X routes map to Draw in Either Half.
- Reversal routes map to GG or Match Over 1.5 after their gates.
- Straight wins require 6 overall wins, 6 relevant split wins and HT/FT
  behaviour confirmation.

Full rule sheet:
PAPASENSE_V1_13_AMENDMENTS.md

ONE-TIME SUPABASE DATABASE STEP
-------------------------------
Open Supabase → SQL Editor and run:

server/supabase/migrations/20260717_v111_accounts_watchlist_push.sql

This is still the correct migration because v1.13 preserves the v1.11 account,
watchlist, notification and pipeline tables.

AUTH SETUP
----------
Render environment variables:
- SUPABASE_URL
- SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- API_FOOTBALL_KEY

Supabase → Authentication → URL Configuration:
Site URL:
https://betspapa.com

Redirect URLs:
https://betspapa.com/account.html
https://www.betspapa.com/account.html

For Google sign-in:
Supabase → Authentication → Providers → Google → Enable
Then add the Google client ID and client secret.

PUSH SETUP
----------
From the server folder:

npm.cmd run vapid

Add the displayed values to Render:
- VAPID_PUBLIC_KEY
- VAPID_PRIVATE_KEY
- PUSH_SUBJECT=mailto:admin@betspapa.com

PIPELINE
--------
The automatic workflow has separate jobs for previous results, today and
tomorrow. Run the normal automatic workflow after deployment so predictions are
regenerated with papasense-v1.13.0 and current team-goal odds.

PWA UPDATE
----------
Installed users receive an Update Now card when the new service worker is ready.
