# BetsPapa v1.16.1 — Faster Picks and Results Repair

## What changed

The v1.16 interface, branded PWA, walkthrough, live states, settlement logic and PapaSense v1.13 rules remain intact. This patch changes how public data is delivered and recovered.

### Faster opening

The Today page stores the latest successful dashboard response in the browser. On a repeat visit, those picks are rendered immediately. A live API request then runs quietly and replaces the saved copy when it succeeds.

The frontend now remembers the last API address that worked and uses a 9-second timeout instead of waiting 35–45 seconds per endpoint. A failed refresh does not clear cards already visible.

### Faster backend responses

Live-score synchronization now runs in the background for ordinary public page requests. The dashboard does not wait for API-Football synchronization before returning Supabase picks. Short in-memory stale-while-revalidate caches reduce repeated database work.

### Results repair

The Results endpoint no longer triggers a live-score refresh before reading results. Large fixture, prediction, team and league ID lists are split into safe Supabase batches. Historical published predictions are included even when they were generated by an older engine version.

Finished statuses supported by the results path are FT, AET, PEN, AWD and WO. The Results page stores its latest successful response locally and shows a clear error with retry guidance when no saved response exists.

## Deployment

No new Supabase migration or Render environment variable is required. Replace the repository with the full v1.16.1 package, push it, confirm `/api/health` reports `1.16.1`, then open `/reset.html` once.
